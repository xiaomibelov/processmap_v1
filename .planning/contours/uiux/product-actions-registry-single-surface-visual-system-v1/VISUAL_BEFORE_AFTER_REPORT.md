# VISUAL_BEFORE_AFTER_REPORT

## Before

- Registry page used multiple disconnected card-like sections.
- Main container and page background used gradient styling.
- Scope looked like a segmented card control.
- Metrics were pill/card-like.
- Filters and warning carried extra framing.
- Table rows looked like individual cards, weakening the table as the primary object.

## After

- Registry content reads as one native ProcessMap working surface.
- Internal sections are separated by thin `#F3F4F6` separators.
- Header, scope tabs, metrics, filters, AI controls, warning and table share one visual system.
- Metrics are text-only and compact.
- AI controls are a restrained row with one purple action.
- Warning is compact text, not a filled alert card.
- Table has a quiet header, light row separators, hover `#FAFAFA`, and status badges as the only strong table colors.
