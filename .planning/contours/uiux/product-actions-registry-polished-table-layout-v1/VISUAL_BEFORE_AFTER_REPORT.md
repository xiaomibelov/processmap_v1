# VISUAL_BEFORE_AFTER_REPORT

## Before

- Header/export/status area competed with the title.
- Metrics looked like pills in a continuous gray sheet.
- Filters were one flat toolbar; applied state was hard to scan.
- AI controls looked like generic load-row controls near session/source context.
- Incomplete warning used a stronger warning treatment than needed.
- Table rows read as separate cards rather than one main working table.

## After

- Header is a defined top band with stronger `Реестр действий с продуктом`, secondary subtitle, compact back action, and single CSV/XLSX export placement.
- Metrics read as a compact dashboard.
- Filters have main/secondary grouping and visible applied-filter affordances.
- AI has a primary action block with secondary chips and adjacent selection counter.
- Warning is softer and includes bounded quick filtering.
- Table uses the available workspace width better, with sticky header, row separators, hover state, and muted BPMN code.
