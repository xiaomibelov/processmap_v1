# Empty- и populated-state expectations по scope

- contour: `uiux/product-actions-registry-noise-cleanup-single-container-v1`
- run_id: `20260518T164643Z-83747`
- источник: PLAN.md §4.13, §4.6–§4.13; UX_SPEC_IMPLEMENTATION_MAP.md §C, §13.

Документ описывает ожидаемое поведение страницы для **каждого scope-таба** при двух исходах: данные присутствуют / данных нет. Все пункты — runtime-проверяемы.

## 0. Общие правила (для всех scope)

- [ ] Структура страницы (Header → Scope tabs → 16px → один белый контейнер → 7 секций) одинакова при пустом и заполненном состояниях.
- [ ] Empty-state не подставляет фейковые строки/метрики/AI-предложения.
- [ ] При смене scope visual-каркас не «прыгает» (контейнер сохраняет размер/радиусы/тень).
- [ ] При просмотре в любом scope нет PUT/PATCH/DELETE в Network.

## 1. Scope = Workspace

### 1.1 Populated (есть проекты и сессии в workspace)

- [ ] Workspace scope (collapsible) свёрнут по умолчанию; строка вида `Workspace scope · N сессий, M строк` с реальными N>0, M>0.
- [ ] Sessions workspace показывает реальные сессии: project/session short ID, UUID, имя проекта, путь, кол-во действий, draft-статус, дата.
- [ ] Кнопки «Открыть проект» (outline) и «Открыть сессию» (`#7C3AED`) активны и кликабельны.
- [ ] Metrics: `2 сессий  152 строк  149 полных  3 неполных  152 после фильтров` — все числа реальные (или соответствующие текущим данным workspace).
- [ ] Filters активны; «Сбросить фильтры» отображается, но при отсутствии применённых фильтров не должен подсвечиваться.
- [ ] Warning row отображается, если в данных есть «Неполные», иначе скрыт.
- [ ] AI suggestions row активен; счётчик `Выбрано для AI: 0 / 10` нейтральный.
- [ ] Таблица заполнена; раскрытие строк работает.

### 1.2 Empty (workspace пуст: нет проектов или нет сессий с действиями)

- [ ] Workspace scope строка показывает `Workspace scope · 0 сессий, 0 строк`.
- [ ] Sessions workspace: пустое состояние без demo-сессий; правая сводка `Всего: 0, без действий: 0` цвета `#9CA3AF`.
- [ ] Metrics: все числа `0`; «неполных» = `0` отображается **нейтральным цветом** (не оранжевым), либо приглушено (`#9CA3AF`), но не `#F59E0B`.
- [ ] Filters остаются в DOM, но в неактивном/задизейбленном состоянии (или активны без эффекта при пустом наборе).
- [ ] Warning row **скрыт** (нет «неполных» — нет повода для warning).
- [ ] AI suggestions row отображается, но кнопка `AI: предложить действия` либо disabled, либо клик ведёт к нейтральному ответу без мутаций. Никаких автоматически сгенерированных «AI-черновиков» в DOM.
- [ ] Таблица показывает корректное пустое состояние (текст вида «Нет данных» / иллюстративное сообщение). **Без** sample-строк.

## 2. Scope = Проект

### 2.1 Populated (выбран проект с действиями)

- [ ] Workspace scope строка пересчитана под выбранный проект: `Workspace scope · N сессий, M строк` (N, M — по проекту).
- [ ] Sessions workspace ограничен сессиями данного проекта.
- [ ] Metrics соответствуют выборке проекта.
- [ ] Filters применимы; «Сбросить фильтры» возвращает все строки проекта.
- [ ] Warning, AI, Table — как в §1.1, но по данным проекта.

### 2.2 Empty (проект без действий / без сессий)

- [ ] Workspace scope строка: `Workspace scope · 0 сессий, 0 строк`.
- [ ] Sessions workspace: пустое состояние.
- [ ] Metrics: `0` по всем числам.
- [ ] Filters могут быть disabled.
- [ ] Warning row скрыт.
- [ ] AI suggestions row виден, кнопка disabled или нейтральна (без авто-мутаций).
- [ ] Таблица — пустое состояние без фейков.

## 3. Scope = Сессия

### 3.1 Populated (выбрана конкретная сессия с действиями)

- [ ] Workspace scope строка показывает агрегаты по выбранной сессии (1 сессия, M строк).
- [ ] Sessions workspace может содержать одну строку с этой сессией (или быть свернута); кнопка «Открыть сессию» ведёт в saved-сессию.
- [ ] Metrics — по сессии.
- [ ] Warning row отображается, если есть «Неполные» в сессии.
- [ ] Таблица заполнена строками сессии; раскрытие показывает 4 read-only поля: ID · BPMN · Сессия · Дата.

### 3.2 Empty (сессия без действий)

- [ ] Workspace scope строка: `Workspace scope · 1 сессии, 0 строк` (или `0 сессий, 0 строк`, если scope = «сессия не выбрана»).
- [ ] Sessions workspace показывает только эту сессию (если выбрана) с `0` действий.
- [ ] Metrics: `0` по числам.
- [ ] Warning row скрыт.
- [ ] AI suggestions: кнопка disabled / нейтральна.
- [ ] Таблица — empty state без фейков.

## 4. Edge cases

- [ ] При смене scope с populated на empty (например, переключение с workspace на пустую сессию) DOM таблицы перестраивается без визуальных артефактов (без «зависших» старых строк).
- [ ] Фильтры, применённые в одном scope, не утекают в другой scope (нет «прилипания» фильтра между табами без явного применения).
- [ ] При loading (асинхронная подгрузка): допустимо отображать нейтральный skeleton/индикатор, но **не** demo-данные.
- [ ] При ошибке загрузки: отображается компактное сообщение об ошибке, без card-style оформления; warning-row при этом не используется как контейнер ошибки.
