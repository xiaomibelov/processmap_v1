# Логика Zoom Threshold

## Цель
Уменьшить сложность отрисовки фигур при низком уровне масштаба.

## Пороги

| Масштаб | Отрисовка фигур | Связи | Метки |
|---------|-----------------|-------|-------|
| `< 0.2` | Только первый `<rect>` (bounding box); все path/circle/polygon скрыты | Без изменений | Оставлены bpmn-js |
| `0.2 – 0.5` | Все `<rect>` видны; path/circle/polygon скрыты | Без изменений | Оставлены bpmn-js |
| `≥ 0.5` | Полная отрисовка | Без изменений | Оставлены bpmn-js |

## Реализация

Функция `simplifyShapeGfx(gfx, scale)` в `cullBpmnViewport.js` манипулирует дочерними элементами внутри `.djs-visual`:
- Находит все `rect`, `path`, `circle`, `polygon`, `ellipse`, `g`
- Устанавливает `style.display = 'none'` на ненужные элементы
- При `scale >= 0.5` сбрасывает `display = ''` для всех

## Почему не пересоздаём SVG

- Пересоздание SVG с нуля требует парсинга XML и потеряет внутренние ссылки bpmn-js
- Манипуляция `display` дочерних узлов безопасна и обратима
