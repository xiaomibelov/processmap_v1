# Спецификация debounce оверлеев

**Контур**: `fix/canvas-overlay-debounce-v1`  
**Запуск**: `20260528T190318Z-4670`

---

## 1. Проблема

При панорамировании bpmn-js генерирует непрерывные события изменения viewbox. Кастомные оверлеи ProcessMap (property labels, status badges, AI indicators) пересчитывают/перемещают позицию на КАЖДОМ событии, вызывая forced reflow 3754 SVG-узлов.

## 2. Решение

Заменить прямое обновление позиций оверлеев на debounced/throttled обновление.

## 3. Что подвергается debounce

| Компонент | Событие | Текущее поведение | Целевое поведение |
|-----------|---------|-------------------|-------------------|
| Property labels | `canvas.viewbox.changed` | Обновление на каждом frame | Debounce 150 мс, trailing |
| Status badges | `canvas.viewbox.changed` | Обновление на каждом frame | Debounce 150 мс, trailing |
| AI indicators | `canvas.viewbox.changed` | Обновление на каждом frame | Debounce 150 мс, trailing |

## 4. Что НЕ подвергается debounce

- Рендеринг фигур bpmn-js (шейперы, connection bendpoints).
- Рендеринг связей.
- Selection highlight.
- Скруббер / minimap.
- Zoom.

## 5. Edge case: inline edit

Если оверлей находится в режиме inline-редактирования:
- Обходить debounce для этого конкретного оверлея.
- Позиция должна обновляться синхронно, чтобы курсор оставался с текстом.

## 6. Технические параметры

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| Задержка debounce | 150 мс | Баланс между отзывчивостью и снижением нагрузки |
| trailing | true | Оверлеи обновляются после остановки пана |
| leading | false | Не обновлять на первом событии (уже идёт плавный пан) |
| rAF throttle (fallback) | каждый 3-й кадр | Если debounce вызывает визуальный лаг |

## 7. Ожидаемый эффект

- Снижение forced reflow на ~80-90% при панорамировании.
- Освобождение CPU для рендеринга фигур bpmn-js.
- Прирост FPS: +8–10 на больших диаграммах.
