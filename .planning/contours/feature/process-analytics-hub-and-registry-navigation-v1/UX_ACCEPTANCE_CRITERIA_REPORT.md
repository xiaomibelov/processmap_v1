## 3. TopBar поведение

- При `?surface=analytics` TopBar показывает `data-testid="topbar-analytics-label"` с текстом «Аналитика».
- Back-кнопка («← Проекты» / «← К проекту») скрыта — не конфликтует с кнопкой «Закрыть» в Hub.
- На обычных экранах (workspace, project, session) back-кнопка работает как раньше.

---

## 4. Registry nesting

- Открытие «Реестра действий» из Hub переходит на `?surface=product-actions-registry`.
- При этом в URL добавляется `return_to=analytics`.
- Закрытие registry считывает `return_to=analytics` и возвращает пользователя обратно в Hub.
- Если `return_to` отсутствует — fallback к workspace (сохранение существующего поведения).

---

## 5. Отсутствие user trap

- Пользователь видит явную кнопку «Закрыть» в Hub.
- Пользователь может использовать browser back.
- Пользователь может кликнуть на навигацию workspace через sidebar.
- Никаких модальных оверлеев без кнопки закрытия.

---

## 6. Console errors

- Единственная ошибка в консоли: `401 Unauthorized` на `/api/auth/me` — это ожидаемое поведение для неавторизованной сессии браузера, не связано с Analytics Hub.
- Ошибок, специфичных для Analytics Hub, не обнаружено.
