# BPMN Editor Improvements — Audit

Контур аудита четырёх задач по BPMN-редактору ProcessMap.

## Задачи

| # | Задача | Статус | Артефакты |
|---|--------|--------|-----------|
| 1 | Version Conflict Detection — оповещение о более новой версии | Аудит | `01-version-conflict/` |
| 2 | Version Merge — подгрузка версии другого пользователя | Аудит | `02-version-merge/` |
| 3 | UI/UX redesign блока троеточие (⋯) | Аудит | `03-ellipsis-menu/` |
| 4 | V2 Overlays checkbox bug | Аудит | `04-v2-overlays/` |

## Общие выводы
- Бэкенд уже имеет механизм версионирования (`diagram_state_version`, CAS, last-write actor).
- Фронтенд уже умеет опрашивать версии и показывать конфликтный модал, но оповещение слишком редкое и подавляется при локальных изменениях.
- Merge-режим требует нового UI с двумя viewers; алгоритмический merge BPMN XML не делаем.
- Меню `⋯` перегружено и дублирует canvas action bar.
- V2 overlays имеют баг в diff-логике и отсутствие cleanup.

## Следующий шаг
Ожидание аппрува аудита перед началом реализации.
