# 5-PLANE — Version Conflict Detection

## 1. UX Plane (пользовательский опыт)
- **Сейчас**: конфликт обнаруживается только при сохранении; пользователь видит модал с кнопками "Обновить / Остаться / Отбросить".
- **Нужно**: неблокирующий индикатор "есть новая версия" в шапке редактора, видимый даже при локальных изменениях; клик открывает детали или merge-режим.
- **Тост**: уже существует `ProcessSaveAckToast` с persistent `remote_update`; можно расширить.

## 2. Data Plane (данные)
- **Сервер** уже хранит `diagram_state_version`, автора и время последней записи.
- **Клиент** уже передаёт `base_diagram_state_version`.
- **Нужен** лёгкий endpoint/поле для проверки актуальной версии без разворачивания XML.

## 3. Logic Plane (логика и состояние)
- `pollRemoteSessionSnapshot` сравнивает `headVersion` и `localVersion`.
- Флаг `localUnsafe` отключает оповещение при несохранённых изменениях.
- **Нужно** разделить "уведомление" и "применение": уведомление показывать всегда, применение/refresh — только когда безопасно.

## 4. Integration Plane (интеграция и производительность)
- Варианты доставки версии:
  - отдельный `GET /api/sessions/{id}/version`;
  - добавить `diagram_state_version` в ответ `POST /api/sessions/{id}/presence`;
  - более частый опрос `meta`.
- Без WebSocket/SSE ограничены request/response; присутствие уже даёт 30-с интервал "бесплатно".

## 5. Security / Auth Plane
- Версию сессии может читать только участник с правом `view`.
- Актёр last-write возвращается из БД; нужно убедиться, что email/имена не утекают пользователям без прав.
