## 4) Draft design — HybridLayer v1

### 4.1 Data model (session meta + localStorage split)
- Session meta (source of truth for layer data):
```json
{
  "bpmn_meta": {
    "hybrid_layer_by_element_id": {
      "Task_1": { "dx": 0, "dy": 0 },
      "Task_2": { "dx": 18, "dy": -10 }
    }
  }
}
```
- UI prefs (per-user, local only):
```json
{
  "by_user": {
    "<user_id_or_anon>": {
      "visible": false,
      "mode": "view",
      "opacity": 60,
      "lock": false,
      "focus": false
    }
  }
}
```
- Storage key: `hybrid_ui_v1` (`hybridLayerUi.js:14`).

### 4.2 Renderer approach (non-invasive overlay)
- Hybrid rendered as separate absolute overlay over BPMN stage (`ProcessStage.jsx` 6562+).
- `Hidden`: overlay not mounted (DOM removed).
- `View`: overlay mounted, hotspot clickable, geometry non-intrusive.
- `Edit`: overlay mounted + shield active, drag allowed; `Esc` -> `View`.
- Overlay click-guard must keep `stopPropagation` and playback guard marker to avoid canvas-side side effects.

### 4.3 Layers UX contract
- Action bar button `Layers` opens popover with:
  - `Hybrid` toggle
  - `Mode`: `View/Edit`
  - `Opacity`: `100/60/30`
  - `Lock`
  - `Focus`
  - Ready/Incomplete counters
  - hint: hold `H` for temporary peek
- Existing anchors: `ProcessStage.jsx` 6172-6280.

### 4.4 Playback integration points (must not reset playback)
- Use existing guard callback `markPlaybackOverlayInteraction(...)` on layers/hybrid interactions.
- Keep one-tick ignore guard semantics for canvas click listeners (already used for overlay interactions).
- No BPMN re-import on layer mode switches.

### 4.5 DOC export integration (proposal for next step)

- Integration point: `frontend/src/components/process/DocStage.jsx` and `buildSessionDocMarkdown` pipeline.
- Minimal v1 for docs:
