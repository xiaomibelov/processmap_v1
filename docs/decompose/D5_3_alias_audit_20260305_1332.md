## Implementation Plan (D5.3)
1. Router registration canonicalization:
   - In `backend/app/routers/_shared.py`, map legacy alias paths to canonical path keys and deduplicate by `(method, canonical_path)`.
   - Prefer canonical declaration when both canonical and alias are present.
2. Compat middleware expansion:
   - In `backend/app/main.py`, extend deprecated alias rules for:
     - sessions `meta` and `bpmn/meta`
     - sessions `ai/ops`
   - Add generic trailing slash compatibility:
     - if `/api/.../` and canonical route exists for same method -> redirect/proxy to stripped path.
3. Keep warnings and headers:
   - `Deprecation`, `Warning`, `Link`, `X-Deprecated-Endpoint`.

## Post-change expectation
- Primary registered routes are canonical-only.
- Legacy clients still work through compat middleware.
- `templates` + `hybrid_stencil_v1` flow remains unchanged.
