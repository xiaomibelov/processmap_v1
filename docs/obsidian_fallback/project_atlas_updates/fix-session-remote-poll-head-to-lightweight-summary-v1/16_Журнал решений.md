# 16_Журнал решений

## ADR-20260430-002 — Remote polling использует lightweight head

status:: accepted
date:: 2026-04-30

### Контекст

Remote poll каждые 9 секунд уже использовал lightweight `GET /api/sessions/{id}/bpmn/versions?limit=1`, но при обнаружении более свежего head автоматически переходил к full `GET /api/sessions/{id}`. Audit baseline показал, что full session payload может быть 4.35 MB / 3.41s.

### Решение

Background poll больше не вызывает full session refresh. Он использует version head для remote indicator/toast. Full session hydration переносится в explicit user refresh action.

| Решение | Деталь |
| ------- | ------ |
| Head truth | `diagram_state_version` из latest BPMN version head |
| Actor label | `author_display`, `author_name`, `author_email`, `created_by` из head |
| Dedup key | `session_payload_hash`, version id, created_at |
| Full session | только explicit refresh |

### Последствия

> [!success] Плюсы
> 9-second poll больше не может незаметно подтянуть multi-MB full session payload.

> [!warning] Ограничение
> До explicit refresh indicator не знает точный список изменённых BPMN element ids, потому что XML намеренно не загружается в фоне.
