# 12_Версии и история

## Remote head poll и history list — разные сценарии

| Сценарий | Endpoint | Payload policy | UI behavior |
| -------- | -------- | -------------- | ----------- |
| Remote update detection | `GET /api/sessions/{id}/bpmn/versions?limit=1` | headers-only, no XML | сравнить head с local `diagram_state_version`, показать indicator |
| History modal list | `GET /api/sessions/{id}/bpmn/versions?limit=N` | headers-only by default | показать список ревизий |
| Version preview/restore | `GET /api/sessions/{id}/bpmn/versions/{version_id}` or explicit include XML flow | XML only for selected version/action | preview/restore |
| Explicit session refresh | `GET /api/sessions/{id}` | full session | user-triggered hydration |

> [!summary] Контракт
> `bpmn/versions?limit=1` достаточно для определения “серверная диаграмма новее локальной”. Он не должен становиться заменой full session hydration и не должен тащить XML/history bodies по умолчанию.
