## ADR-20260501-002 - Secondary session PATCHes resolve CAS base at send time

status:: accepted
date:: 2026-05-01

### Контекст

Несколько frontend writers используют `PATCH /api/sessions/{id}`. Delayed writer мог отправить stale `base_diagram_state_version` после same-client accepted write.

### Решение

| Механизм | Решение |
| -------- | ------- |
| Ordering | per-session promise queue |
| Base | resolved at send time через `getBaseDiagramStateVersion` |
| Success | accepted `diagram_state_version` remembered |
| 409 | `server_current_version` remembered, response remains failed |
| Scope | diagram/session truth secondary PATCH writers |

### Последствия

> [!success] Same-client self-conflict снижен
> PATCH B отправляется после accepted PATCH A и берёт updated base.

> [!warning] No blind retry
> Coordinator не делает retry и не превращает external conflict в success.
