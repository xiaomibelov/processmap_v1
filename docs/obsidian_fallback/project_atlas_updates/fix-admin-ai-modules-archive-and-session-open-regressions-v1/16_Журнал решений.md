# 2026-05-07 - fix/admin-ai-modules-archive-and-session-open-regressions-v1

Decision:

- Archiving an active code-seeded prompt is an explicit Admin action and must not be undone by idempotent seed logic.
- Code seed is allowed to create missing default prompts, but it must not reactivate a prompt that exists and is archived.
- Admin prompt registry setup failures are operational errors and should be returned as controlled responses.
- Workspace/session open actions must pass full session context where available; session id alone is not enough for first-click URL/navigation correctness.
- Project open and session open affordances stay distinct: `Открыть проект` for projects, `Открыть сессию` for sessions.
