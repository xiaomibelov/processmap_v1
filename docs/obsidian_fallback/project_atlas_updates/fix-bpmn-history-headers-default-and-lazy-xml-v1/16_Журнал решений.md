## ADR-20260501-001 - История BPMN-версий: headers-only list и lazy XML detail

status:: accepted
date:: 2026-05-01

### Контекст

Performance audit показал повторные запросы `GET /api/sessions/{id}/bpmn/versions?limit=50` на initial session load. Также существовал риск использовать `include_xml=true` для bulk history list, хотя XML нужен только для выбранной версии.

### Решение

Список истории грузится как headers-only. Head/badge path использует `limit=1`. XML загружается лениво через single-version detail endpoint при preview/restore/explicit detail action.

### Последствия

| Последствие | Оценка |
| ----------- | ------ |
| Initial load | Меньше лишних history list запросов |
| History modal | Делает один headers-only list fetch |
| Preview | Делает отдельный lazy detail fetch |
| Restore | CAS/write semantics unchanged |
| Backend | Contract unchanged |

> [!success] Принцип
> History list, remote head и XML detail остаются разными контрактами.
