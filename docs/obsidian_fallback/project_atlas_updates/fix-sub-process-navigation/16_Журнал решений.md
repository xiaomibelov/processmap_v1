## ADR: навигация в подпроцессы и размещение breadcrumb

### 1. Drill-down только через стрелку `.bjs-drilldown`

**Решение:** навигация в подпроцесс срабатывает только по клику на стандартную drilldown-стрелку bpmn-js. Одиночный клик по телу элемента оставлен для выделения.

**Причина:** bpmn-js предоставляет встроенную overlay-стрелку, которая очевидна для пользователя и не конфликтует с selection/overlay pipeline.

**Реализация:** `bindSubprocessNavigationEvents` слушает клики на `.bjs-drilldown` в capture-фазе, находит соответствующий `type: "drilldown"` overlay и вызывает `onNavigateToSubprocess(element.id)`.

### 2. Breadcrumb поверх канваса

**Решение:** breadcrumb подпроцесса вынесен из топ-бара внутрь `.workspaceMain` как абсолютный overlay.

**Причина:** освобождает вертикальное пространство под хедером, убирает пустоту и предотвращает схлопывание канваса.

**Реализация:** `SubprocessBreadcrumbs` рендерится только при `breadcrumbs.length >= 2` внутри `.subprocessBreadcrumbsOnCanvas`.

### 3. Отступ breadcrumb 12px

**Решение:** панель breadcrumb отодвинута от верхнего и левого края рабочей области на `12px`.

**Причина:** визуальная полировка — панель не должна прилипать к хедеру и краям канваса.

**Реализация:** `.subprocessBreadcrumbsOnCanvas { top: 12px; left: 12px; }`.

### 4. Layout-сетка `appTopStack`

**Решение:** top-элементы обёрнуты в `.appTopStack`, для `.appRoot` используется `grid-rows-[auto_minmax(0,1fr)_auto]`.

**Причина:** канвас должен занимать всё оставшееся пространство независимо от высоты топ-блока.

**Реализация:** `frontend/src/components/AppShell.jsx` + `frontend/src/styles/tailwind.css`.
