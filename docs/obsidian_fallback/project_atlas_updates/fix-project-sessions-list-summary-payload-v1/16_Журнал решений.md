# 16_Журнал решений

## ADR-20260430-001 — Лёгкий summary contract для списка сессий проекта

status:: accepted

context::

| Факт | Значение |
| ---- | -------- |
| Bottleneck | `GET /api/projects/e41791f9f7/sessions` |
| Before | `4,803,702 bytes / 4448ms` |
| Причина | Full session rows через `_session_api_dump()` и `Storage.list()` / `SELECT *` |

decision::

> [!summary]
> Добавить additive query mode `view=summary` для `GET /api/projects/{id}/sessions` и org-scoped аналога. Frontend bootstrap использует summary по умолчанию, full detail остаётся через bare endpoint, `view=full` и `GET /api/sessions/{id}`.

consequences::

| Плюс | Минус / ограничение |
| ---- | ------------------- |
| Initial project session list больше не тянет multi-MB session blobs | Stage proof pending до deploy |
| Existing full contract не сломан | Summary не заменяет full session truth |
| Открытие session явно грузит full detail по выбранному id | Explorer-specific endpoint остаётся отдельным contour |
