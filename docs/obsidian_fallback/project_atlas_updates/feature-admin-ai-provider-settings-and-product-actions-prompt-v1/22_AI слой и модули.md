  - `GET /api/admin/ai/provider-settings`
  - `POST /api/admin/ai/provider-settings`
  - `POST /api/admin/ai/provider-settings/verify`
- Provider summary remains secret-safe: frontend sees `has_api_key`, `base_url`, provider metadata and verify result only; existing API key is never returned.
- `ai.product_actions.suggest` has a code-seeded global active prompt version `seed_ai_product_actions_suggest_v2`.
- Seed `seed_ai_product_actions_suggest_v1` is archived so already-seeded environments can advance to the specific product-actions prompt.
- Product-actions AI setup failures now return controlled codes:
  - `AI_PROVIDER_NOT_CONFIGURED`
  - `AI_PROMPT_NOT_CONFIGURED`
  - `AI_PROVIDER_ERROR`
- Apply/save path remains review-first: product action suggestions are not domain truth until accepted through the existing `interview.analysis.product_actions[]` patch path.
