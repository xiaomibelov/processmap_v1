# 2026-05-07 - feature/admin-ai-provider-settings-and-product-actions-prompt-v1

Decision:

- DeepSeek provider management belongs under Admin -> AI modules, not in the global header or product-actions surface.
- Admin provider endpoints must be secret-safe and return `has_api_key`, never the stored key.
- `ai.product_actions.suggest` requires an active prompt from registry after seed; if registry cannot provide one, endpoint returns `AI_PROMPT_NOT_CONFIGURED`.
- Product-actions provider setup failures are expected operational states and should return controlled error codes, not raw provider/stack failures.
- Product actions AI remains suggestion-only; no auto-write to `interview.analysis.product_actions[]`.
