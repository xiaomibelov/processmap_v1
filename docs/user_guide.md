# Руководство пользователя PROCESSMAP

## 1. Что это и зачем
PROCESSMAP помогает собирать и формализовать процесс в одном рабочем контуре:
- интервью с командой,
- BPMN-диаграмма,
- XML,
- контроль качества и покрытия,
- версии схемы.

Ключевая ценность: изменения в Interview/Diagram/XML синхронизируются в рамках одной сессии процесса.

## 2. Быстрый старт (3–5 минут)
1. Откройте `/`.
2. Нажмите `Войти` или перейдите на `/login`.
3. Авторизуйтесь (`admin@local` / `admin` в dev-конфигурации).
4. В TopBar нажмите `Новый проект`.
5. В модалке создайте проект; сессия создаётся автоматически.
6. Выберите вкладку `Diagram` и начните редактировать схему.
7. Откройте левую панель (`Открыть панель`) для AI-вопросов, заметок и покрытия.

## 3. Основные сущности
- `Проект`: контейнер процесса и сессий.
- `Сессия`: рабочий экземпляр с диаграммой, интервью и заметками.
- `Узел`: элемент диаграммы (шаг, событие, развилка).
- `Акторы/Lanes`: роли процесса.
- `Interview`: таблично-формовый режим уточнения шагов.
- `DoD/Coverage`: признаки заполненности данных по узлам.

## 4. Разделы интерфейса
## 4.1 Публичная часть
- `/`: лендинг с CTA `Войти` и `Регистрация`.
- `/login`: страница входа.
- Модальное окно логина вызывается из лендинга.

## 4.2 Рабочая зона `/app`
- `TopBar`: проект/сессия, создание, удаление, AI статус, выход.
- `Tabs`: `Interview | Diagram | XML | DOC`.
- `Diagram toolbar`: сохранение, режимы отображения (`Normal/Interview/Quality/Coverage`), overflow-меню, `Требует внимания`.
- `Left Sidebar`: выбранный узел, AI-вопросы, заметки, акторы, шаблоны/TL;DR.

## 5. Режимы диаграммы
Режим хранится в `localStorage` по ключу `ui.diagram.mode.v1`.

- `Normal`: чистая диаграмма, минимум оверлеев.
- `Interview`: оверлеи интервью (AI/notes/readiness).
- `Quality`: список проблем качества схемы, человеко-читаемые описания, действие `Показать на схеме`.
- `Coverage`: карта пробелов (нет AI/заметок/других полей), minimap и jump-to.

## 6. Кнопки и действия
Полный каталог с условиями, эффектами и API: `docs/ui_actions_catalog.md`.

## 7. Типовые сценарии
## 7.1 Создать процесс и первую сессию
1. Войти.
2. `Новый проект`.
3. Заполнить название и нажать `Создать`.
4. Проверить, что проект и сессия выбраны в TopBar.

## 7.2 Проверить качество схемы
1. Перейти на `Diagram`.
2. Включить режим `Quality`.
3. В списке проблем нажать `Показать на схеме`.
4. Проверить центрирование и фокус узла.

## 7.3 Найти пробелы в данных
1. Перейти в режим `Coverage`.
2. Открыть `Требует внимания`.
3. Фильтровать по `Quality/AI/Notes`.
4. Нажать `Показать на схеме` для перехода к узлу.

## 7.4 Работа с заметками узла
1. Выбрать узел на диаграмме.
2. Открыть левую панель → `Заметки`.
3. Вкладка `К узлу`, ввести текст, `Сохранить заметку`.

## 7.5 Генерация AI-вопросов
1. Выбрать узел.
2. В левой панели открыть `AI-вопросы`.
3. Нажать `Сгенерировать вопросы`.
4. Отметить статусы, добавить комментарий, `Сохранить`.

## 8. Частые проблемы
- `401 missing_bearer`: невалидный/просроченный access token; требуется повторный вход.
- `Пустой список сессий`: проверьте выбранный проект.
- `Не видно узел в сайдбаре`: выберите элемент на диаграмме; lane/пул не считаются задачами.
- `Падение e2e браузера в текущем окружении`: см. `docs/screens/README.md`.

## 9. Глоссарий
- `Шаг`: действие процесса (task).
- `Переход`: стрелка между шагами.
- `Развилка`: точка ветвления.
- `Coverage`: степень заполненности данных по узлам.
- `Quality`: набор проверок корректности схемы.

## 10. How to verify (smoke)
Рекомендуемый smoke-проход:
1. Логин.
2. Создание проекта/сессии.
3. Выбор узла на диаграмме.
4. Генерация AI-вопросов.
5. Добавление заметки.
6. Открытие Quality и `Показать на схеме`.
7. Reload страницы и проверка сохранения состояния.

Команды, которые запускались в этом цикле проверки:
- `cd frontend && E2E_BROWSER=webkit E2E_APP_BASE_URL=http://127.0.0.1:5177 E2E_API_BASE_URL=http://127.0.0.1:8011 npx playwright test e2e/auth-routing-login.spec.mjs --workers=1 --reporter=list`
- `cd frontend && E2E_BROWSER=chromium E2E_APP_BASE_URL=http://127.0.0.1:5177 E2E_API_BASE_URL=http://127.0.0.1:8011 npx playwright test e2e/auth-routing-login.spec.mjs --workers=1 --reporter=list`

Оба запуска завершились ошибками browser launch (`SIGTRAP/Abort trap`) в текущем окружении.
