## Description
All e2e tests that seed BPMN XML via `PUT /api/sessions/{sid}/bpmn` fail with:
```
DIAGRAM_STATE_BASE_VERSION_REQUIRED
client_base_version: null
server_current_version: 0
```

The backend `session_bpmn_save` endpoint now requires `base_version` in the request body (optimistic locking / CAS guard). The e2e test fixtures do not provide this field.

## Affected tests
- e2e/tab-transition-matrix-big.spec.mjs
- e2e/bpmn-roundtrip-big.spec.mjs
- e2e/ai-button-opens-panel-and-shows-loading.spec.mjs
- e2e/manual-snapshots-accumulate.spec.mjs
- e2e/snapshot-versions-big-diagram-restore.spec.mjs

## Root cause
Backend `_legacy_main.py` lines ~950 and ~5750 require `base_version` param. E2e `apiJson` helper in tests does not send it.

## Fix options
1. E2e fixtures: send `base_version: 0` or `base_version: session.version` on initial seed
2. Backend: allow `base_version: 0` or `base_version: null` for new sessions (first write)
